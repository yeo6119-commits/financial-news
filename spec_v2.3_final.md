# 금융회사 디지털·AI 뉴스 수집 시스템 — v2.3 최종 확정 사양

**확정일**: 2026-07-09
**핵심 원칙**: 수집은 넓게, 제외·분류·요약은 본문 확보 후 로컬에서 판단. 어떤 기사도 조용히 사라지지 않는다.

---

## 1. 시스템 개요

두 개의 소스에서 이원 수집한다.

| 소스 | 방식 | 역할 |
|---|---|---|
| 뉴스 | 네이버 뉴스 API 전수 수집 | 메인 그물 — 언론 보도 전체 |
| 보도자료 | 9개 금융그룹 홈페이지 직접 크롤링 | 선행 소스 — 언론보다 빠르거나 언론이 안 다루는 1차 발표 확보 |

수집 → 본문 추출 → 필터 → 중복 제거 → 분류 → 요약(기사당 1회) → HTML 아카이브 → delivered 트랜잭션.

**운영비 0원**: 네이버 API 무료 + Groq 무료 티어 + GitHub Actions/Pages 무료.

## 3. 검색 전략 (뉴스)

**A. 앱·브랜드명 단독 검색** — 하나원큐, KB스타뱅킹, 슈퍼쏠, 모니모, 영웅문 등
**B. 회사명 × 서비스 키워드 조합** — Tier 1은 8개(앱·디지털·AI·인공지능·생성형AI·플랫폼·비대면·MTS), Tier 2는 4개(앱·디지털·AI·플랫폼)
**C. 지주·그룹명 × 전략 키워드** — AI·인공지능·디지털·플랫폼·데이터·생성형AI

- 노이즈 브랜드(쏠·원큐·리브·WON·올원)는 단독 검색 제외, **본문 매칭 전용**
- 리뉴얼·개편·고도화·출시·도입 등은 검색어가 아닌 **본문 관련성 판단 키워드**
- 표기 변형 병기: 신한SOL/신한 쏠/슈퍼쏠, iM/DGB, JP모건/제이피모건 등

## 4. 검색 시간 규칙

| 상황 | 검색 범위 |
|---|---|
| 기본 | `last_successful_run_at − 60분` ~ 요청 시각 (오버랩 = 인덱싱 지연 보험) |
| 첫 실행 | 최근 24시간 |
| 월요일 오전 + 실행기록 없음/실패 | 직전 금요일 13:00부터 |
| 72시간 초과 | 최근 72시간만 자동 수집, **잘린 구간 시각을 HTML·DB에 명시** (수동 재수집 지시 가능) |

## 5. 네이버 API 수집 루프

- sort=date, display=100, start 1→1000 페이지네이션
- 기간 필터는 루프 안에서 pubDate 기준 수행
- **페이지 전체가 기간 이전일 때만 중단 + 1페이지 추가 확인** (pubDate 정렬 불완전 대응)
- start=1000 도달 시에도 기간 내 기사 존재 → **수집 절단 플래그** + 절단 키워드 통계 표시
- API 호출 수 계측 (일 한도 25,000, 예상 회당 1,500~3,000콜)
- 레이트리밋(429) 지수 백오프 재시도

## 6. 보도자료 수집 (press_collector — 9개 그룹 어댑터)

| 그룹 | 방식 | 비고 |
|---|---|---|
| 하나 | POST ajax(moreList.do) → HTML 조각 | 접근 확인 완료. 상세 view.do?hnewsSn={id}, 계열사는 #해시태그 |
| KB | GET JSON API (notics?bulbdId=8) | **라이브 검증 완료** (3,757건). 계열사는 제목 추출 폴백 |
| 신한 | JSON API | GET 시 405 확인 — 구현 시 POST/헤더 재확인, 그전까지 실패 허용 모드 |
| 우리 | HTML list.do / view.do | **legacy SSL 옵션 필요** (OP_LEGACY_SERVER_CONNECT) |
| NH | HTML 게시판 | |
| BNK | 정적 HTML | 회사명 = 제목 접두사 |
| JB | 정적 HTML | 회사명 = 제목 접두사 |
| 메리츠 | POST pr_search.do | http 평문 — https 우선 시도 |
| iM | POST nbbsR1.fg | 본문이 목록 응답에 포함(PUTUP_WRIT_CN), 날짜 RGDT |

**운영 원칙**:
- 실행마다 어댑터별 헬스체크 → 실패 시 HTML 통계에 "PR 소스 실패: ○○" 표시 (조용한 유실 금지)
- 페이지네이션은 검색 기간 이전 도달 시 조기 중단
- 한국투자는 WAF로 제외. 2차 후보: 카카오뱅크·토스·케이뱅크 뉴스룸, 삼성금융
- URL·파라미터는 2026-05~06 확인값 — 구현 시 재검증

## 7. 본문 추출

1순위 네이버 뉴스 링크(n.news.naver.com, 구조 일관) → 2순위 originallink → 실패 시에도 기사 보존.
보도자료는 각 어댑터의 상세 URL 패턴 사용 (하나는 JS 링크라 URL 패턴 별도 처리).

## 8. 필터링

- 제외: 스포츠 / 범죄 / 금융 디지털·AI 무관 / 단순 홍보·이벤트
- **보안 오버라이드**: 보이스피싱·FDS·스미싱·인증 강화 등은 범죄 키워드에 걸려도 디지털 관련성 재판단 (하나원큐 스미싱 서비스 = 수록 대상)
- 무관 예시: AI 기업 지원 '대출상품'은 디지털 채널 기사 아님 → 무관 처리하되 제외 목록에 사유 명시
- **모든 제외는 삭제가 아니라 표시** — HTML 하단 제외 목록에서 사유와 함께 검수 가능

## 9. 분류

- 금융그룹(메뉴) → 업권 → **디지털 | AI**
- AI 키워드: AI·인공지능·생성형AI·LLM·대규모언어모델·AI 에이전트·에이전틱·머신러닝·딥러닝
- 제목/본문에 AI 키워드 있으면 AI, 그 외 디지털

## 10. 중복 제거

| 비교 대상 | 기준 |
|---|---|
| 이번 배치 내 | URL·제목·본문 해시 + 제목 유사도(0.85) + simhash(해밍 ≤3) → 대표 기사 1건 |
| 과거 delivered | **URL·정규화 제목 완전 일치만** (후속 보도 보호) |
| 보도자료 ↔ 뉴스 | 유사 시 보도자료 카드에 언론 기사 링크 병기 (둘 다 보존) |

제목 정규화: [단독]·[속보] 등 태그 제거, 공백·특수문자·대소문자 통일, 언론사 접두어 제거.
대표 기사 선정: 본문 추출 성공 > 본문 길이 > 언론사 신뢰도 > 최초 보도. Fintechtoday 등 비권위 소스 제외 유지.

## 11. 요약 (Groq 무료)

- **기사 1건 = LLM 1회 호출, 배치 요약 금지** (config에 batch:false 고정)
- 모델: Groq Llama 3.3 70B (유실 원인이던 배치 처리 제거로 안전, 품질 불만 시 provider 한 줄 교체)
- 형식: * 로 시작하는 3줄, -함/-임체 임원 보고용, 약어 첫 등장 시 괄호 풀이, 본문에 없는 내용 추정 금지
- 검증(정확히 3줄·각 줄 * 시작·최소 길이) 실패 시 지수 백오프 재시도 3회
- **최종 실패해도 [요약 실패] 카드로 HTML 수록** — 로그에만 남기지 않음

## 12. HTML 아카이브

- 실행마다 **DB 60일치 전체를 단일 페이지로 재생성** (`financial_news_archive.html` 고정 이름)
- 상단: 컴팩트 수지 라인 (수집 = 반영 + 기간외 + 중복 + 무관, 실패·절단 표시)
- **고정 회사 메뉴 12개 탭** (전체 + 11 카테고리, 건수 배지, 클릭 필터)
- **회차(요청 일시)별 타임라인** — 탭 선택 시 그 회사 기사가 날짜순으로 쭉
- 기사 카드: [그룹/업권/디지털·AI] 태그 + [보도자료] 소스 배지 + 제목 + 언론사·일시 + 3줄 요약 + 매칭 키워드 + **하단 원문 보기 링크 버튼**
- 실패 기사: [요약 실패]/[본문 추출 실패] 상태로 해당 카테고리에 수록
- 하단: 접이식 제외 목록 (사유별)

## 13. delivered 트랜잭션 (유실 방지 핵심)

HTML 임시 파일 생성 → 성공 확인 → rename → **그 후에만** DB 트랜잭션으로 delivered=1 + last_successful_run_at 갱신 → commit.
역순 절대 금지 (delivered 먼저 찍고 HTML 실패 = 영구 유실).

## 14. DB

- SQLite, 60일 보관, 본문 전문 미저장 (해시·fingerprint·3줄 요약만)
- 60일 초과분 매 실행 자동 정리

## 15. 운영 환경

| 단계 | 환경 |
|---|---|
| 초기 구축·튜닝 (1~2주) | **맥북** — 로컬 실행으로 노이즈율·추출 실패율·요약 품질 디버깅 |
| 배포 | GitHub 저장소(private) + Actions 스케줄/수동 실행 + Pages 게시 |
| 일상 사용 | **아이폰·아이패드** — GitHub 앱에서 실행 버튼, Safari에서 아카이브 URL 열람 |

세팅 순서: Python 3.11+ → venv → `pip install requests beautifulsoup4 lxml pyyaml python-dotenv` → 네이버 API 키 + Groq 키 발급 → .env 작성 → 로컬 실행 검증 → GitHub 업로드.

## 17. 검증된 사실 (파일럿에서 확인)
