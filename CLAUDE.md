# 금융 뉴스 수집 시스템 — 작업 규칙

구조·아키텍처는 코드와 `docs/`를 읽어 파악할 것. 이 파일은 **코드만 봐서는 알 수 없는 함정과 판단 근거**만 담는다.

---

## 절대 규칙

**1. 요약 배치 금지** — `config.yaml`의 `batch: false`를 `true`로 바꾸지 말 것.
기사를 묶어 요약하면 토큰은 아끼지만 모델이 일부 기사를 조용히 누락시킨다. 초기 기사 유실의 원인으로 진단됐다. 기사 1건 = Groq 호출 1회.

**2. VACUUM은 트랜잭션 밖에서** — `db.py`의 `cleanup()`에서 이 순서를 지킬 것.
```python
conn.commit()
conn.isolation_level = None
conn.execute("VACUUM")
conn.isolation_level = ""
```
이 수정이 **5번 날아갔다.** `db.py`를 새로 쓸 때마다 빠뜨렸기 때문. 워크플로 초록불의 전제 조건이다.

**3. push 전 `./check.sh`** — 반복 재발한 수정이 살아 있는지 검사한다. 실패하면 고치고 다시. 실제로 구버전이 최신 파일을 덮어쓴 사고를 잡아냈다.

**4. 제외 기사도 보존** — 필터에서 걸러도 사유와 함께 HTML에 표시한다. "어떤 기사도 조용히 사라지지 않는다"가 이 시스템의 제1원칙.

---

## 필터에 넣으면 안 되는 것

**`sports_exclude`에 일반어 금지** — "경기"를 넣었다가 **경기도 기사가 전멸**한 사고가 있었다.
- 경기 → 경기도, 경기 전망
- 시즌 → 시즌 프로모션
- 리그 → 리그오브레전드
- 우승 → 공모전 우승
- 전반기/후반기 → 반기 실적

KBO·탈삼진·시청률처럼 **스포츠에만 쓰이는 말만** 넣는다.

**`KB` 단독 키워드 금지** — `KBO`가 매칭되어 야구 기사가 들어온다. 또 브랜드 목록에 들어가면 관련성 키워드로 자동 편입되어 모든 KB 기사가 무조건 통과한다. `KB금융`, `KB국민은행` 등 정식 명칭만.

**본문 판정에 앱 이름 금지** — `BODY_CORE`(주제어급 49개)만 쓴다. 관련성 목록 전체(122개)를 쓰면 CSR 기사에 "신청은 하나원큐 앱에서" 한 줄만 있어도 통과한다. 삼계탕 나눔 기사가 이렇게 뚫렸다.

**`standalone_mode` 그룹에 대기업 본사 금지** — 인터넷·핀테크는 회사명이 그대로 검색어가 된다. "네이버"를 넣으면 웹툰·쇼핑·스포츠를 전부 긁어와 실행이 13분으로 늘고 절단이 발생한다.

**네이버 검색은 토큰 단위 매칭** — `"신한 SOL"`은 `신한`만으로도 매칭되어 1,000건 상한(절단)에 걸린다. 회사명+일반어 조합 대신 `SOL LINK`, `쏠뱅크` 같은 **고유명**을 쓴다.

## 필터 수정 시 순서 주의

`prescreen()`에서 **관련성 검사가 `NON_DIGITAL_TOPICS` 검사보다 먼저**여야 한다.
- `하나은행, AI 기반 금리 산정` → '금리'가 있어도 AI 때문에 통과 ✓
- `신한은행, 디지털 취약계층 AI 교육` → '취약계층'이 있어도 디지털 때문에 통과 ✓

순서를 바꾸면 정상 기사가 잘린다.

## 요약 (Groq)

**프롬프트 예시의 밀도가 출력 밀도를 결정한다.** 토큰 아끼려고 예시를 34자로 압축했더니 모델이 13자짜리 요약(`은행 자금 주식매매 연동`)을 냈다. 예시는 실제 원하는 밀도(50~70자)로 유지할 것.

**근거 검증(`_grounded`)을 제거하지 말 것.** 모델이 요약할 내용을 못 찾으면 **프롬프트의 예시를 그대로 복창한다.** 골드만삭스 기사와 JP모건 기사에 똑같은 "KB AI Lab" 요약이 나온 사고가 있었다. 요약 토큰의 55% 이상이 원문에 있어야 통과.

**형식 실패는 1회만 재시도.** 같은 프롬프트로 다시 불러도 같은 결과다. 3회 재시도 + 8b 3회 = 최대 6회는 **한 건에 정상 6건치 토큰**을 태운다.

**토큰의 70%가 본문.** `body_chars`가 하루 처리량을 결정한다. 1100자 → 70b 하루 52건. 무료 티어는 모델당 일일 10만 토큰, 한국시간 오전 9시 리셋.

## 중복 제거

**본문 simhash만으로는 안 잡힌다.** 한국 기사는 같은 사건이어도 해밍거리가 14~28로 벌어진다. **제목 토큰 겹침**이 주력이다.

**조사 제거 필수** — `공무원연금공단`과 `공무원연금공단과`가 다른 토큰으로 갈려 중복을 놓쳤다. `_strip_josa()`를 건드리지 말 것.

## 검증 습관

필터·중복 규칙을 바꾸면 **반드시 회귀 테스트**를 돌린다. 정상 기사가 함께 잘리는지 확인해야 한다. 정상 기사 샘플:

```
KB금융, 실전형 AI 인재 양성 프로그램 KB AI Lab 출범
NH농협은행, AI 웰니스 솔루션 'HEALLO' 100여 개 지점 도입
은행·증권 한 계좌로…신한투자증권 SOL LINK 흥행
토스 카드라운지, 출시 1년 만에 방문자 11배↑
```

제외돼야 할 샘플:
```
[인사] 한국산업은행
[여의도단신] 키움증권·미래에셋자산운용·한국투자신탁운용·KB자산운용
하나증권 "삼미금속, AI 데이터센터 전력 인프라 공급망 진입"
은행주, 증시 급락 속 강세…케이뱅크 14%대 급등
하나금융, 어르신에 삼계탕 나눔 활동 진행
```

## 운영

- **실행은 웹페이지 [실행] 버튼 또는 GitHub Actions.** 로컬에서 `main.py`를 돌리지 말 것 — `news.db`가 원격과 충돌한다.
- **`news.db`는 `.gitignore`.** Actions만 `git add -f`로 커밋한다.
- **반영 0건이면** 수지 라인의 흐름을 볼 것. `1차통과 정상 → 중복제거후 0`이면 기열람(정상), `1차통과 0`이면 필터 버그.
- **`추출실패 0 · 요약실패 0`은 오해를 부른다.** `live`(중복 제거 후) 기준이라 반영이 0이면 자동으로 0이 된다.

---

## 배경

사용자는 국내외 금융회사의 디지털·AI 뉴스를 모니터링한다. 임원 보고용이 아니라 **사실 파악용**이라, 요약은 해석·전망 없이 본문 사실만 담는다.

자세한 판단 근거는 `docs/decisions.md`, 필터 규칙 전체는 `docs/filtering.md` 참조.
