# 필터링 규칙

가장 자주 손보는 부분. 오탐 기사가 보이면 이 문서를 보고 어느 관문에서 뚫렸는지 짚는다.

---

## 두 관문

| 관문 | 함수 | 판단 근거 | 규모 |
|---|---|---|---|
| **1차 스크리닝** | `filter.prescreen()` | 제목만 | ~1,800 → ~300 |
| **2차 필터** | `filter.apply_filters()` | 본문 리드 400자 | ~300 → 정제 |

1차에서 본문 추출 대상을 줄이는 게 핵심이다. 본문 추출은 건당 1~5초라, 안 걸러내면 실행 시간이 21분까지 늘어난다 (실제로 그랬던 이력이 있다).

---

## 1차 스크리닝 순서

위에서부터 순서대로 검사하고, 걸리면 즉시 제외한다.

### (0) 주식·애널리스트 리포트 — 44개 패턴

```
목표가, 목표주가, 컨센서스, 투자의견, 매수의견, 실적 전망, 주가 조정,
저평가, 고평가, 적정주가, 애널리포트, 주목할 종목, 특징주, 종목진단,
주가, 은행주, 금융주, 증시, 상한가, 신고가, 52주, 시총, 코스피, 코스닥,
지분 확대, 외인 지분, 공매도, 배당수익률,
투자 노하우, 고점, 저점, 레버리지, 조정 국면, 약세장, 강세장 ...
```

**증권사 리포트 정규식** — 증권사명 뒤에 바로 따옴표가 오면 종목 분석이다.
```python
STOCK_REPORT_RE = re.compile(r'(증권|금융투자)\s*[,]?\s*["\u201c\u2018\']')
```
> `하나증권 "삼미금속, AI 데이터센터 진입"` → 제목에 AI가 있어도 제외.
> `신한투자증권, SOL LINK 계좌 출시` → 따옴표 없음 → 통과.

**투자 코너 태그**
```python
INVEST_TAG_RE = re.compile(r"^\s*\[(투자|머니|재테크|증시|마켓|스탁|stock|주식)", re.I)
```

### (0-b) 인사·부고

```python
PERSONNEL_RE = re.compile(
    r"^\[(인사|부고|동정|화촉|승진)\]|\[인사\]|임원\s?인사|정기\s?인사|"
    r"인사\s?발령|승진\s?인사|^\[신임\]|부고\s?:")
```

> "인사"는 "인사이트", "인사말"에도 들어간다. 단순 문자열 매칭은 위험해서 **대괄호 태그와 명시적 표현만** 정규식으로 잡는다.

### (0-c) 태그 + 구분자 3개 이상 → 종합기사

```python
TAG_RE = re.compile(r"^\s*[\[\【]")
SEP_RE = re.compile(r"[·ㆍ／/]")
if TAG_RE.match(title) and len(SEP_RE.findall(title)) >= 3:
    → 제외
```

**왜 구분자로 세는가**: 태그 이름(`[주목! e금융]`, `[여의도단신]`, `[금융레이더]`)은 매체마다 계속 새로 생겨서 열거 방식이 한계가 있다. 또 회사명이 `하나`, `KB국민`, `IBK기업`처럼 잘려서 매칭이 안 되기도 한다. **구조로 판정**하는 게 근본적이다.

| 제목 | 구분자 | 판정 |
|---|---|---|
| `[주목! e금융] 하나·신한은행·케이뱅크·KB국민·IBK기업...` | 4 | 제외 |
| `[여의도단신] 키움증권·미래에셋자산운용·한국투자신탁운용·KB자산운용` | 3 | 제외 |
| `[단독] 신한·하나은행, AI 공동개발` | 1 | 통과 |
| `NH농협은행, AI·디지털 전환 가속` | 1 (태그 없음) | 통과 |

### (1) 묶음 기사

```python
BUNDLE_PATTERNS = [
    r"^\[.{0,10}(브리핑|풍향계|이모저모|24시|톡톡|NOW|픽|소식|동향|종합|단신|레이더)\]",
    r"^\[(오늘의|데일리|주간|금융|증권|보험|카드|여신)",
    r"外\s*$", r"\.\.\.\s*外", r"…\s*外",
]
```

- 회사명이 **하나도 없으면** → 주체 불명 → 제외
- 태그 기사에 회사가 **2곳 이상** → 종합기사 → 제외
- 태그 없이 회사가 **4곳 이상** → 종합기사 → 제외
- `[뉴스워커_하나카드] 판다 디자인 트래블로그` → 1곳 → **통과** (단일 회사 코너물)

`_dedup_companies()`로 부분문자열 관계는 하나로 센다 (`KB금융`과 `KB국민카드`가 함께 잡혀도 1개).

### (2) 업종 무관 노이즈 — 31개

퀴즈·이벤트·홈쇼핑·면세점·게임·바둑 등.

### (3) 스포츠 — 30개

```
축구, 야구, 농구, 배구, 골프, 월드컵, K리그, 프로야구, KBO, 올림픽,
구단, 홈런, 탈삼진, 타자, 투수, 안타, 실책, 시청률, 빅매치, 개막전,
데뷔전, 국가대표, 아시안게임, e스포츠, 감독 선임, 선수단, 치어리더 ...
```

> **금지**: "경기", "시즌", "리그", "우승", "전반기/후반기", "선수", "감독"
> 경기도·경기전망·시즌프로모션·공모전우승·반기실적 기사가 잘린다. 실제 사고 이력 있음.

### (4) 범죄 — 단, 보안 오버라이드 우선

```yaml
security_override: [보이스피싱, 이상거래탐지, FDS, 이상금융거래,
                    해킹, 스미싱, 딥페이크, 개인정보 유출, 정보보호]
```

> `금융사기 예방`, `인증 강화`는 **뺐다.** 오프라인 노인 대상 예방교육 기사가 통과하던 원인.

### (5) 회사명 필수

제목에 대상 금융사가 없으면 제외. 회사 키워드는 총 **229개**.

### (6) 관련성 키워드 — 없으면 본문 확인으로

관련성 키워드는 총 **122개**:
- `search_service_keywords` + `local_relevance_keywords`
- `classification.ai_keywords`
- `filters.security_override`
- `EXTRA_RELEVANCE` (23개)
- **`BRAND_TO_COMPANY`의 앱·서비스명 전체** ← 앱 이름이 제목에 있으면 그 자체로 디지털 기사

**있으면** → 즉시 통과
**없으면** → (7)로

### (7) 본문 읽을 가치 판단 — `NON_DIGITAL_TOPICS` 63개

제목에 디지털 단서가 없으면서 아래 주제면, **본문을 읽지 않고** 제외한다.

| 분류 | 키워드 |
|---|---|
| 사회공헌 | 나눔, 봉사, 기탁, 후원, 성금, 헌혈, 김장, 연탄, 장학, 삼계탕, 취약계층, 복지관 ... |
| 금리·상품 | 금리, 예금, 적금, 특판, 청약, 대출 상품, 보험료 ... |
| 실적·재무 | 순이익, 영업이익, 당기순, 실적 발표, 결산, 배당, 자사주 ... |
| 행사·조직 | 개점, 지점 신설, 채용, 공모전, 위촉, 간담회, 창립, 골프대회 ... |

**순서가 중요하다.** 관련성 검사를 먼저 하므로:
- `하나은행, AI 기반 금리 산정` → '금리'가 있어도 AI 때문에 **통과**
- `신한은행, 디지털 취약계층 AI 교육` → '취약계층'이 있어도 디지털 때문에 **통과**

### (8) 나머지 → `_needs_body_check = True`

본문 추출 후 2차 필터에서 판정.

---

## 2차 필터 — 본문 리드 판정

```python
if article.get("_needs_body_check"):
    lead = body[:400]                      # 리드만
    core_hits = _hit(lead, body_core_keywords(cfg))   # 핵심 키워드만
    if not core_hits:
        → 제외 "무관(본문 확인: 디지털·AI 내용 아님)"
```

### 왜 리드 400자인가

한국 기사는 역피라미드 구조라 주제가 리드에 온다. 뒤쪽에 스치듯 나오는 언급은 그 기사의 주제가 아니다.

### 왜 핵심 키워드만인가

관련성 목록 전체(122개)를 쓰면 **앱 이름까지 포함**되어, CSR 기사에 "신청은 하나원큐 앱에서" 한 줄만 있어도 통과한다. 실제로 삼계탕 나눔 기사가 이렇게 뚫렸다.

`BODY_CORE` (43개) + `ai_keywords` = **49개**만 인정:
```
AI, 인공지능, 생성형, LLM, 머신러닝, 딥러닝, 챗봇, GPT, 에이전트,
디지털, DT, DX, 플랫폼, 빅데이터, 클라우드, 블록체인, 스테이블코인,
가상자산, 디지털자산, 핀테크, 오픈뱅킹, API, 자동화, 무인, 비대면,
시스템 구축, 고도화, 전산, IT, 알고리즘, 로보어드바이저, MTS, FDS ...
```

## 오탐이 보일 때 진단법

### 1. HTML 수지 라인을 본다

```
수집 1823 = 반영 0 + 제외 1823 · 흐름: 수집 1823 → 1차통과 312 → 중복제거후 0 → 반영 0
· 제외 사유: 기열람 298, 무관 1401, 스포츠 52, 종합기사 41
```

| 증상 | 원인 |
|---|---|
| 1차통과 0 | 필터 버그 — 회사 키워드 목록이 비었을 가능성 |
| 1차통과 정상, 중복제거후 0 | **정상** — 오늘 이미 돌려서 다 기열람 |
| 스포츠·무관이 비정상적으로 많음 | 키워드에 일반어가 섞임 |

### 2. `diagnose.py` 실행

```bash
python diagnose.py
```
회사·관련성 키워드 개수와 실제 기사 판정을 보여준다. 키워드가 0개면 config 구조가 깨진 것.

### 3. 제외 목록 패널 확인

HTML 하단에서 제외 사유를 직접 본다.

## 규칙 추가 요령

| 상황 | 위치 |
|---|---|
| 특정 워딩이 계속 들어옴 | `filter.py`의 해당 패턴 리스트 |
| 특정 코너물이 계속 들어옴 | `BUNDLE_PATTERNS` 또는 구조 규칙 검토 |
| 회사가 안 잡힘 | `config.yaml`의 tier1/tier2 |
| 앱 이름이 안 잡힘 | `classifier.py`의 `BRAND_TO_COMPANY` |

**추가 전에 회귀 테스트를 돌릴 것.** 정상 기사가 함께 잘리는지 확인해야 한다. "경기"를 스포츠에 넣었다가 경기도 기사가 전멸한 사고가 있었다.
